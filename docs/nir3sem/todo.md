# Миграция на S3 MinIO

1. Пользователь сабмитит решение в webserver
2. webserver кладёт код решения на S3
3. webserver регистрирует решение в БД со ссылкой на S3
4. webserver кладёт в kafka сообщение на проверку решения
5. test-runner достает сообщение из kafka и скачивает решение и задачу из S3
6. test-runner запускает тесты для решения
7. test-runner кладёт весь вывод на S3
8. test-runner кладёт в kafka сообщение о результатах проверки тестов
9. webserver достает сообщение из kafka и сохраняет результаты проверки в БД

### **Рекомендуемая стратегия: `at-least-once` + idempotent processing**

#### **1. На стороне webserver (producer)**
- Используйте **idempotent producer** в Kafka (`enable.idempotence=true`). Это гарантирует, что одно и то же сообщение не будет записано в топик дважды при повторных попытках отправки.
- При сабмите решения:
    - Генерируйте **уникальный `solution_id`** (например, UUID или составной ID из `user_id + task_id + timestamp`).
    - Храните `solution_id` в БД как часть сущности `Submission`.
    - Это `solution_id` включайте в Kafka-сообщение.

#### **2. На стороне test-runner (consumer)**
- Запись результата на S3 также должна быть идемпотентной:
    - Используйте `solution_id` как часть пути: `s3://results/{solution_id}/output.log`.
    - Повторная запись по тому же пути не нарушит консистентность.

#### **3. На стороне webserver (consumer результатов)**
- При получении результата из Kafka используйте `solution_id` для идемпотентного обновления записи в БД.
- Если запись уже существует — игнорируйте или логируйте дубликат (но не ломайте систему).

#### **4. Настройка Kafka**
- `enable.idempotence=true` у producer’ов.
- Consumer: `enable.auto.commit=false`, commit offset **после успешной обработки и сохранения результата**.

---

### **Итоговая архитектурная стратегия**

| Компонент                  | Стратегия                               | Механизм реализации                               |
|----------------------------|-----------------------------------------|---------------------------------------------------|
| Webserver (submit → Kafka) | Идемпотентная отправка                  | Idempotent producer + `solution_id`               |
| Test-runner                | Идемпотентная обработка                 | Проверка по `solution_id`, UPSERT в БД            |
| Webserver (result ← Kafka) | Идемпотентное сохранение                | UPSERT в БД по `solution_id`                      |
| Kafka                      | `at-least-once` (с idempotent producer) | `enable.idempotence=true`, ручной коммит оффсетов |
