В системе должны храниться и обрабатываться следующие основные сущности:
- пользователь;
- привилегия;
- роль;
- задача;
- тег;
- решение;
- результат проверки;
- язык.
Для каждой сущности должны быть определены обязательные и необязательные атрибуты, связи между сущностями и ограничения целостности данных.
Минимальный состав основных данных должен включать:
- Пользователь: идентификатор, никнейм, email, логин, пароль в хэшированном виде, привилегии, роли, дата регистрации;
- Задача: идентификатор, название, описание, автор, сложность, теги, языки, шаблон решения, авто-тесты, эталонное решение;
- Тег: идентификатор, название;
- Решение: идентификатор, задача, автор, язык, код;
- Результат проверки: идентификатор, решение, код завершения, число тестов, число провалов, число ошибок, засчитано ли решение;
