# Репликация сервисов

Система должна быть надежной и доступной.
Для доступности необходимо, чтобы при падении одного инстанса сервиса, система не стала недоступной.
Как бонус, это повышает количество обратываемых запросов системой.

Для репликации сервиса web-server добавил Redis с общим состоянием подписок на получение результата.
Сервис test-runner и ранее позволял репликацию.

# LSP для редактора кода

Для js/ts есть встроенная мощная поддержка редактирования.
Для остальных языков есть лишь подсветка и слабый автокомплит.
Я добавил запуск lsp серверов на бэкенде в docker контейнерах.
В результате пользовательский опыт намного-намного лучше, чем было,
и даже чем на всех других платформах.
Можно даже положить в образ библиотеки и использовать их при решении задач.

# Генерация задачи с помощью AI агента

К форме редактирования задачи я добавил возможность сбросить форму в виде файлов на диск в директорию.
В этой директории пользователь запустит opencode/claudecode/qwencode или что-то похожее.
Подключит WebMCP платформы к агенту.
Подключит skill для работы с задачами к агенту.
Попросит агента выполнить какую-то работу с задачей.
Агент выполнит работу согласно скиллу и выполнит валидацию через WebMCP.
Отгрузит задачу обратно на фронт. Там пользователь ее сохранит.

Такая архитектура позволяет не изобретать заново функционал кодинговых агентов и не организовывать перепродажу токенов,
если бы генерация происходила на бэкенде.
