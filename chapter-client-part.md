# Конструирование клиентской части

Клиентская часть веб-приложения реализована на базе фреймворка React с использованием языка TypeScript для обеспечения типобезопасности и поддержки IDE. Архитектура построена на компонентном подходе, позволяющем переиспользовать готовые элементы интерфейса. Страницы располагаются в директории `frontend/src/pages`, где каждая поддиректория соответствует отдельному маршруту приложения. Управление состоянием реализовано через библиотеку TanStack Query, а для навигации используется React Router.

## Главная страница

Главная страница (Landing) служит точкой входа на платформу и формирует первое впечатление пользователя. Компонент отображает крупный заголовок «Достигайте мастерства через вызов функции» с градиентным оформлением текста. Под заголовком располагается кнопка «Начать работу», которая перенаправляет аутентифицированных пользователей на страницу выбора задач, а неавторизованных — на страницу авторизации через Keycloak.

Ниже представлен информационный блок, описывающий возможности платформы: практика программирования через небольшие упражнения, поддержка нескольких языков программирования. Для визуализации архитектуры системы используется библиотека ReactFlow, отображающая интерактивную диаграмму связей между компонентами — Frontend, Backend, Kafka и Tester. Диаграмма позволяет перетаскивать узлы и демонстрирует поток данных между сервисами.

## Страница списка задач

Страница Problems предназначена для каталогизации и поиска доступных задач. Интерфейс разделён на две области: панель фильтрации слева (одна треть ширины) и список задач справа. Панель фильтрации содержит следующие элементы управления: текстовое поле для поиска по названию, выпадающий список сложности (Easy, Normal, Hard), мультивыбор языков программирования, мультивыбор тегов. Все фильтры отображаются в строке URL, что обеспечивает возможность сохранять и передавать ссылки на отфильтрованные выборки.

Список задач представлен карточками, где каждая карточка содержит название задачи, индикатор сложности в виде многоугольного бейджа, присвоенные теги и поддерживаемые языки. При клике на название осуществляется переход на страницу решения задачи. Для отображения статистики решаемости задач используется кольцевая диаграмма DonutChart из библиотеки Mantine Charts, показывающая соотношение нерешённых и решённых задач по категориям сложности.

Верхняя часть области списка содержит карусель с тематическими подборками задач: «Задачи на C++», «Задачи на массивы», «Рекурсия». Каждый слайд представляет собой фоновое изображение с текстовым заголовком, при клике на который устанавливаются соответствующие фильтры. Карусель работает в бесконечном режиме и прокручивается по три слайда за раз.

## Страница решения задачи

Страница Problem реализует основной рабочий процесс пользователя — написание и тестирование решения. Интерфейс построен с использованием библиотеки react-resizable-panels, позволяющей пользователю изменять размеры областей перетаскиванием разделителей. Горизонтальная раскладка включает панель описания (по умолчанию 30% ширины) и панель редактора кода.

Панель описания содержит вкладки «Описание» и «Решения». Вкладка описания отображает условие задачи в формате Markdown. Вкладка решений показывает историю попыток пользователя с результатами тестирования. Для выбора языка программирования используется сегментированный переключатель, отображающий доступные варианты с иконками языков.

Редактор кода реализован на базе Monaco Editor, который обеспечивает подсветку синтаксиса, автодополнение и другие возможности профессиональной IDE. При смене языка редактор автоматически загружает шаблон решения для выбранного языка. Пользователь может сбросить код к шаблону через всплывающую подсказку. Кнопка «Запустить» инициирует отправку кода на сервер для выполнения тестов.

Дополнительно реализована функция совместной работы в реальном времени через WebSocket. При активации режима «Онлайн» пользователь получает уникальную ссылку для приглашения других участников. Совместное редактирование реализовано с использованием библиотек Yjs для совместного состояния и y-monaco для привязки к Monaco Editor.

## Страница песочницы

Страница Playground представляет собой онлайн-компилятор для экспериментов с кодом без привязки к конкретной задаче. Основной элемент страницы — редактор кода с поддержкой IntelliSense, реализованный через Monaco Editor с VSCode API. Пользователь может выбрать один из трёх поддерживаемых языков: C++17 (gcc 13.2.0), Java 21 (Corretto) или Node.js 20. Переключение языка обновляет содержимое редактора с примером кода на выбранном языке.

Вывод результатов выполнения отображается в правой панели. Система использует Server-Sent Events (SSE) для получения логов в реальном времени. Сообщения классифицируются по трём типам: обычный вывод (чёрный текст), ошибки выполнения (красный текст), информационные сообщения (синий текст). Статус выполнения отображается в заголовке страницы при помощи бейджа.

Интерфейс поддерживает горячую клавишу F9 для быстрого запуска кода. Все настройки расположения панелей сохраняются в localStorage благодаря автоматическому сохранению состояния библиотеки react-resizable-panels.

## Страница списка своих задач и создания задачи

Страница MyProblems отображает таблицу задач, созданных текущим пользователем. Таблица содержит колонки: идентификатор, название, сложность, теги, поддерживаемые языки и действия. В колонке действий доступны кнопки редактирования и удаления. Удаление выполняется через мутацию с автоматическим обновлением списка после завершения запроса. Кнопка «Создать задачу» в верхней части страницы перенаправляет пользователя на форму создания.

Страница MyProblem реализует форму создания и редактирования задачи. Верхняя часть содержит навигационные элементы и кнопки управления: подключение директории проекта, импорт и экспорт описания задачи в формате Markdown. Подключение директории использует File System Access API, позволяя сохранять данные непосредственно на диск пользователя.

Форма состоит из двух основных блоков. Первый блок — левая панель с настройками задачи: название и мультивыбор тегов. Второй блок — редактор описания задачи на всю оставшуюся ширину, реализованный через Monaco Editor. Ниже располагается секция настройки языковых реализаций, где пользователь может добавить поддержку нового языка через выпадающий список. Для каждого добавленного языка открывается вкладка с тремя редакторами кода: тесты, эталонное решение и шаблон решения. Все данные формы хранятся в локальном состоянии и отправляются на сервер при нажатии кнопки «Сохранить».
